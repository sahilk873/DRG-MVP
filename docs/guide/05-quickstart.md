# 5 · Quickstart & verification

Get from clone to "I can see it work" fast. Requires **Python 3.11+** and **Node 22+**.
Everything here is deterministic and needs **no model credential** (the extraction agent is
the only model-dependent piece, and it's optional for the demo/CLI).

## One-time setup

```sh
python -m venv .venv && source .venv/bin/activate && python -m pip install -e .
cd agent && npm ci && cd ..
cd demo  && npm ci && cd ..
```

Console scripts (after `pip install -e .`): `revenue-integrity` (CLI, `cli.py`) and
`revenue-integrity-ingest` (bulk profiler/adapter runner, `ingestion/cli.py`).

## The five gates (what CI runs)

`make verify` is the full local gate. Individually:

| # | Gate | Command |
|---|------|---------|
| 1 | Compile | `python -m compileall -q src` |
| 2 | Python unit tests (527) | `PYTHONPATH=src python -m unittest discover -s tests` |
| 3 | Demo-fixture parity | `make demo-packet-check` (= `python scripts/generate_demo_packet.py --check`) |
| 4 | Agent | `cd agent && npm run check` (node --test + `tsc --noEmit`) |
| 5 | Demo | `cd demo && npm run test && npm run typecheck && npm run build` |

Run a single Python test:

```sh
PYTHONPATH=src python -m unittest tests.test_engine.<Class>.<test>
```

## Run the clinical care-gap engine on the DFU episode

Reproduce the worked example from [How it works](02-how-it-works.md):

```sh
PYTHONPATH=src python - <<'PY'
import json
from revenue_integrity import models, rules, engine, grouper
case = models.EncounterCase.from_dict(json.load(open('examples/case_diabetic_foot_ulcer_episode.json')))
pkg  = rules.RulePackage.from_dict(json.load(open('rules/wound_care_gaps_v1.json')))
findings = engine.RuleEngine(pkg, grouper.DeterministicDemoGrouper()).evaluate(case)
for f in findings:
    print(f.rule_id, "|", f.gap_domain, "|", f.alert_urgency, "|", f.recommended_action)
PY
```

Expected: `CG-INF-002` (delayed_action, urgent), `CG-DET-001` (deterioration on day-28
growth), `CG-DFU-001` (routine), `CG-DFU-002` (same_day).

## Run the extraction agent (optional — needs a model)

The agent is provider-agnostic via `MODEL_ID`; it imports no provider SDK.

```sh
cd agent && cp .env.example .env
MODEL_ID=anthropic/<model> npm run extract -- \
  ../examples/source_bundle_pressure_injury.json ../output/encounter-case.json \
  ../src/revenue_integrity/data/wound_care_ontology_v1.json
```

## Regenerating fixtures (after an engine/packet/automation change)

The demo fixtures are generated by the **canonical** script — never hand-edit them:

```sh
PYTHONPATH=src python scripts/generate_demo_packet.py          # regenerate
PYTHONPATH=src python scripts/generate_demo_packet.py --check  # prove no drift (gate 3)
```

Because the review packet and automation plan are **versioned public contracts**, change the
schema, the Python, the demo Zod, the generated fixture, and the docs **together**, then run
`make demo-packet-check`.

## Governance reminders (from `CONTRIBUTING.md`)

- **Version everything governed** — bump ontology/rule/schema/packet/automation versions on
  any breaking contract change; a governed package must be `approved`/`approved-for-demo`.
- **Test matrix per change type** — rule changes need positive + negative + contradictory +
  malformed-input tests; reader/adapter changes need malformed-file/schema-drift/etc. tests.
- **Preserve** deterministic IDs, integer-cent money, complete version provenance, the
  hash-chained audit, and the claim-mutation wall. **Never** commit PHI or credentials.
