# Setting up this project from the zip — read me first

This archive is a **point-in-time snapshot of the meaningful source** of *Encounter Revenue
Integrity* (the DRG-MVP project). It is **not** a full git clone: to keep the file small, the
large, regenerable, and machine-local directories were left out. This doc tells you exactly
what's here, what isn't, and how to rebuild the rest so everything runs.

> Everything shipped here (rules, codes, prices, grouper, patient data) is **synthetic** —
> a reference implementation, **not for production billing or clinical use**.

---

## What's in the zip

| Path | What it is |
|------|------------|
| `src/revenue_integrity/` | The deterministic Python engine (models, rules, engine, ontology, automation, routing, workflow, review packet, audit, ingestion). |
| `agent/src/` | The Mastra/TypeScript extraction agent (semantic extraction only). |
| `demo/src/` | The React/Vite demo app (renders engine output; revenue + care-gap lenses). |
| `docs/` | All documentation — **start with [`docs/guide/README.md`](docs/guide/README.md)**. |
| `schemas/` | Versioned public JSON-Schema contracts. |
| `rules/`, `examples/` | Governed rule packages + synthetic example cases/episodes. |
| `tests/` | The Python test suite. |
| `scripts/` | Utilities (incl. `generate_demo_packet.py`, the canonical fixture generator). |
| root config | `pyproject.toml`, `Makefile`, `README.md`, `CLAUDE.md`, `CONTRIBUTING.md`, `agent/package.json`, `demo/package.json`, tsconfigs, etc. |

## What was excluded (and how to get it back)

These were left out on purpose — they are large and/or fully regenerable from what's in the zip:

| Excluded | Why | How to restore |
|----------|-----|----------------|
| `.git/` | VCS history; this is a snapshot | Clone the real repo from source control if you need history |
| `agent/node_modules/`, `demo/node_modules/` | JS dependencies (large) | `npm ci` in each (see below) |
| `.venv/` / `venv/` | Python virtualenv | recreate it (see below) |
| `demo/dist/`, build output, `__pycache__/`, `*.pyc`, caches | Build/compile artifacts | regenerated by the build/test commands |
| local tool dirs (`.claude/`, `.mcp.json`, `.playwright-mcp/`, `code-review-graph/`) | machine-/tool-local, not product | not needed to run the project |

## Prerequisites

- **Python 3.11+**
- **Node 22+**

## Rebuild & run (from the unzipped directory)

```sh
# 1. Python engine
python -m venv .venv && source .venv/bin/activate && python -m pip install -e .

# 2. Agent dependencies
cd agent && npm ci && cd ..

# 3. Demo dependencies
cd demo && npm ci && cd ..

# 4. Verify everything (the full local gate)
make verify
```

`make verify` runs the five gates: `compileall` + the Python unit tests + the demo-fixture
parity check + the agent check (`npm run check`) + the demo test/typecheck/build. Individual
gate commands are listed in [`docs/guide/05-quickstart.md`](docs/guide/05-quickstart.md).

## See it work (no model credential needed)

```sh
make demo                 # deterministic CLI demo
cd demo && npm run dev     # the interactive pitch app (revenue + care-gap lenses)
```

Reproduce the Diabetic Foot Ulcer care-gap worked example:

```sh
PYTHONPATH=src python - <<'PY'
import json
from revenue_integrity import models, rules, engine, grouper
case = models.EncounterCase.from_dict(json.load(open('examples/case_diabetic_foot_ulcer_episode.json')))
pkg  = rules.RulePackage.from_dict(json.load(open('rules/wound_care_gaps_v1.json')))
for f in engine.RuleEngine(pkg, grouper.DeterministicDemoGrouper()).evaluate(case):
    print(f.rule_id, "|", f.gap_domain, "|", f.alert_urgency, "|", f.recommended_action)
PY
```

## (Optional) Run the extraction agent

The agent is provider-agnostic via a `MODEL_ID` string and imports no provider SDK. Only this
step needs a model + API key; the engine, demo, and CLI do not.

```sh
cd agent && cp .env.example .env
MODEL_ID=anthropic/<model> npm run extract -- \
  ../examples/source_bundle_pressure_injury.json ../output/encounter-case.json \
  ../src/revenue_integrity/data/wound_care_ontology_v1.json
```
